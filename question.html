1. What is HTML?

    1. HTML stands for HyperText Markup Language, and it's the standard language used to create webpages.
    2. It provides the basic structure of a webpage by organizing content like text, images, links, and more.
    3. HTML uses tags to define different elements on a page.
    4. It is not a programming language, but a markup language that structures content rather than performing logic.
    5. Web browsers read and interpret HTML to display content in a readable and visually structured format.
    6. HTML is usually used along with CSS and JavaScript to style pages and add interactive features.

  2. What is Difference Between HTML and XHTML?

    3. HTML

    1. More flexible and forgiving with coding errors.
    2. Tags are not case-sensitive.
    3. Some tags can be left unclosed (e.g., br, li).
    4. Attribute values can be unquoted in some cases.
    5. Allows looser structure, so pages often still work with mistakes.

    XHTML

    1. Follows strict XML rules, requiring well-formed code.
    2. Tags must be lowercase.
    3. All tags must be properly closed(eg., br tag).
    4. All attribute values must be quoted (e.g., width="100").
    5. Does not tolerate errors—browsers may fail to render invalid code.

  3. What is Tag in HTML?

    1. A tag in HTML is a special keyword used to mark and organize content on a web page.
    2. Tags are placed inside angle brackets like p, h1, or img.
    3. Most tags appear in pairs:
      • Opening tag: p
      • Closing tag: /p
    4. Some tags are self-closing, meaning they don’t require a separate end tag (e.g., br,img).
    5. Tags help the browser understand how to display different types of content—such as text, images, links, or headings.
    6. In short, HTML tags are the core building blocks that define the layout and content structure of a webpage.

  4. What is an Element in HTML?

    5. An HTML element is a complete structure made up of a start tag, some content, and usually an end tag.

    • The element begins with an opening tag, like p.
    • The content you want to display goes between the tags. For example: This is a paragraph.
    • Most elements are closed with a closing tag, such as /p.
    • Some elements are empty, meaning they don’t have any content or a closing tag (like img / or /br).
    • HTML elements are used to describe the purpose and layout of content on a webpage.

  5. What is Difference Between HTML4 and HTML5?

    6. HTML4

    • Released in the late 1990s and was widely used for many years.
    • Requires plugins like Flash to play audio and video.
    • Mainly designed for desktop browsers, not mobile-friendly.
    • Has only basic HTML tags without semantic meaning.
    • Supports only simple form input types like text and password.
    • No built-in support for offline storage or advanced web features.

    HTML5

    • Released in 2014 as an updated standard for modern web development.
    • Supports native audio and video playback without plugins.
    • Built to work smoothly on mobile devices and desktops.
    • Introduces semantic tags like header,footer, and section.
    • Offers new and advanced form input types like email, date, and color.
    • Includes offline storage and APIs for creating web applications.

  6. What are Lists?

    7. Definition

    In HTML, a list is used to arrange a set of related items in an organized format.It improves the clarity and structure of content on a web page.

    Purpose of Lists:

    • To group related information together.
    • To present steps, options, or definitions clearly and neatly.
    • Commonly used in navigation menus, instructions, glossaries, and outlines.

    Types of Lists

    HTML provides three main types of lists

    1. Ordered list
      • Displays items in a specific sequence.
      • Uses numbers or letters (e.g., 1, 2, 3 or a, b, c).
      • Ideal for step-by-step instructions, rankings, or ordered tasks.
    2. Unordered List
      • Shows items in no particular order.
      • Uses bullet points or symbols.
      • Best for simple collections like shopping lists or category menus.
    3. Description List
      • Used for terms and their definitions.
      • Terms go inside dt (definition term), and explanations inside dd (definition description).
      • Perfect for glossaries, FAQs, or concept explanations.

  7. What is Difference Between HTML and XML?

    8. HTML

    • HTML is used to design and display content on web pages.
    • It focuses on how information appears to users.
    • HTML uses predefined tags like p, div, and img.
    • It is not case-sensitive (e.g., BODY and body are treated the same).
    • Browsers can ignore small errors in HTML code.
    • HTML is mainly used in web development and page formatting.

    XML

    • XML is used to store and transport data in a structured format.
    • It focuses on the meaning and organization of data.
    • XML allows users to create their own custom tags.
    • It is case-sensitive (e.g., and are different).
    • XML requires strict rules; errors must be fixed for the document to work.
    • XML is commonly used in data sharing between applications or systems.
